# 抖音后台抓取流程

> 入口：`抓后台.bat` / `扫码登录抖音.bat`（项目根目录双击即可）
>
> 工具脚本：`tools/fetch_douyin_backend.py`
>
> 持久化登录态：项目根目录 `.auth/`（本地扫码后生成，已被 `.gitignore` 排除）

## 日常使用

1. **拉最新视频列表**（覆盖 `context/my-history-backend.md`）

   ```bat
   抓后台.bat
   ```

2. **拉最新视频列表 + 最近 2 条视频的评论**

   ```bat
   抓后台.bat --with-comments
   ```

   评论按点赞排序，写到 `context/comments/<aweme_id>.md`。

3. **拉最新视频列表 + 最近 5 条视频的评论**

   ```bat
   抓后台.bat --with-comments --top 5
   ```

4. **多抓一些历史视频（默认 30 条）**

   ```bat
   抓后台.bat --limit 50
   ```

## 登录态过期处理

抖音 cookie 大约 **3-5 天** 没用会过期，表现：
- `抓后台.bat` 跑完提示 `[失败] 视频列表为空`
- `.cache/videos.json` 里 `status_code: 8` 或 `aweme_list: null`

处理：

```bat
扫码登录抖音.bat
```

会弹一个 Chromium 窗口，用**手机抖音 App** 扫窗口里的二维码。

**注意**：`ensure_login` 函数检测 cookie 名 `sessionid`，实测扫码登录后 cookie 名可能不同会触发 `[登录] 超时未检测到登录态`——**这是误报**，cookies 实际已经持久化到 `.auth/`。直接关掉窗口，再跑一次 `抓后台.bat` 验证就行。

## 一切落到的文件

| 文件 | 内容 |
|---|---|
| `context/my-history-backend.md` | 视频后台摘要（含标题、播放、章节摘要）。每次抓取覆盖 |
| `context/comments/<aweme_id>.md` | 单条视频评论按点赞排序 |
| `.cache/videos.json` | 原始 videos.json（debug 用，可删） |

## 技术细节（出问题时看）

- Python 环境：推荐使用本项目 `.venv/Scripts/python.exe`，并运行 `python -m playwright install chromium`
- 抓取逻辑：`tools/douyin_session/crawler.py`
  - `fetch_recent_videos` 走 `creator.douyin.com/creator-micro/content/manage` 的 work_list 接口
  - `fetch_comments` 走 `www.douyin.com/video/<aid>` 前台滚动加载
- `tools/fetch_douyin_backend.py` 启动时把 cwd 切到项目根目录，这样 crawler 的 `auth_dir()` 会使用本项目 `.auth/`
- 终端中文乱码：bat 已经 `set PYTHONUTF8=1`；如果手动跑 python 出现乱码也是因为 PowerShell stdout 是 GBK
