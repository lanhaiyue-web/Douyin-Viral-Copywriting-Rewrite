# 飞书集成（可选）

> **不用飞书也能跑完整功能**——本地 Streamlit 网页 `start_app.bat` 就够用。
>
> 只有当你想「躺在床上用手机飞书发一句 → 电脑自动写稿 → 推回飞书读」时，才配置飞书。

按这个步骤走，5-10 分钟拿到 App ID + Secret，填到凭证文件就能跑。

## 0. 先决定要不要装 Claude Code / Codex

飞书是可选的，Claude Code / Codex 也是可选的。

- 只想用本地网页：不用装飞书，也不用装 Claude Code / Codex。
- 只想让飞书做评分：可以只填 `DEEPSEEK_API_KEY`，用 **直接 API 运行**。
- 想在飞书里点 **Claude Code 运行** / **Codex 运行**：先装下面两个 CLI。

```powershell
# Claude Code
npm install -g @anthropic-ai/claude-code
claude --version

# Codex CLI
npm install -g @openai/codex
codex --version
codex login
```

首次运行会要求登录你自己的 Claude / OpenAI 账号。登录凭证不在项目目录里，不会被 git 上传。

如果 `start_bot.bat` 提示找不到命令，可以在 `%APPDATA%\baokuan-rewrite\.env` 里指定绝对路径：

```ini
CLAUDE_CMD=C:\你的路径\claude.cmd
CODEX_CMD=C:\你的路径\codex.cmd
```

## 1. 注册并创建自建应用

1. 浏览器打开 [open.feishu.cn](https://open.feishu.cn)，用飞书账号登录
2. 右上角点 **开发者后台** → 进入开放平台
3. 左上角 **创建应用** → 选 **自建应用**
4. 填写应用信息：
   - 应用名称：随便取，比如"文案改写助手"
   - 应用描述：随意
   - 上传一个图标（任意图都行）
5. 点 **创建**

## 2. 拿到 App ID + App Secret，填到凭证文件

1. 进入新创建的应用 → 左侧菜单 **凭证与基础信息**
2. 复制 **App ID** 和 **App Secret**
3. 新建文件 `%APPDATA%\baokuan-rewrite\.env`（Windows），填入：

   ```ini
   FEISHU_APP_ID=cli_xxxxxx
   FEISHU_APP_SECRET=xxxxxxxxxxxx
   DEEPSEEK_API_KEY=sk-xxxxxx
   ```

> **不要把这些值填到项目里的 `.env`**。项目级 `.env` 只放非敏感配置（模型名、CLI 路径），真实 key 一律放 `%APPDATA%\baokuan-rewrite\.env`，这样脱敏 / 删项目 / 上传 git 都不会泄露。

## 3. 开启机器人能力

1. 左侧菜单 **添加应用能力** → 找到 **机器人** → 点 **添加**
2. 添加完成后左侧会出现 **机器人** 菜单项

## 4. 开启长连接事件 / 回调订阅（关键，省掉公网）

飞书这里容易混淆：**接收消息**是"事件"，**卡片按钮点击**是"回调"。它们在同一个左侧菜单下，但通常分属两个 tab。

### 4.1 事件配置：接收用户消息

1. 左侧 **事件与回调** → **事件配置**
2. 点订阅方式旁边的编辑按钮，选择 **使用长连接接收事件**（推荐方式）—— 选这个就不用公网域名
3. 点 **添加事件**，搜并订阅：
   - **接收消息 v2.0**（`im.message.receive_v1`）

### 4.2 回调配置：接收卡片按钮点击

1. 仍在左侧 **事件与回调**，切到 **回调配置** tab
2. 点订阅方式旁边的编辑按钮，选择 **使用长连接接收回调**
3. 点 **添加回调**
4. 在弹窗里选 **卡片** 分类，勾选：
   - **卡片回传交互**（`card.action.trigger`）

如果在"事件配置"里搜不到"卡片回传交互"，这是正常的；请切到"回调配置"再找。

## 5. 申请权限

1. 左侧 **权限管理** → **API 权限**
2. 搜索并申请以下权限（点"开通"）：
   - `im:message` —— 接收消息
   - `im:message:send_as_bot` —— 以机器人身份发消息
   - `im:resource` —— 读取附件（后续上传文件用）

## 6. 发布版本（个人开发者可直接发给自己）

1. 左侧 **版本管理与发布** → **创建版本**
2. 版本号填 `1.0.0`，更新说明随意写
3. 可用范围：**仅可被指定人员使用**，把自己加进去
4. 提交申请 → 个人开发者会自动通过

## 7. 把机器人加到与自己的会话

1. 打开手机或电脑端 **飞书 App**
2. 进入 **通讯录** → 搜索你刚创建的应用名（比如"文案改写助手"）
3. 点 **添加** 或者 **关注**
4. 与机器人开始私聊

## 8. 启动 Bot

电脑端双击 `start_bot.bat`，看到：

```
[启动] App ID: cli_xxxx...
[就绪] 等待飞书消息……
```

就 OK 了。

## 9. 测试

在飞书私聊里发：

```
/help
```

如果回复了使用说明，就连通了。

然后试试：

```
/原创 30岁还没存款的人最该做什么
```

原创 / 改写任务会弹卡片让你选 [Claude Code 运行] / [Codex 运行]；评分任务还会额外显示 [直接 API 运行]。

## 调试清单（常见问题）

**Q: 飞书提示找不到机器人？**
A: 第 6 步发布版本时确认"可用范围"包含自己，且已提交申请。个人开发者发布后等 1-2 分钟生效。

**Q: 发消息没反应？**
A: 看 `start_bot.bat` 的控制台输出，有没有报错。检查 `%APPDATA%\baokuan-rewrite\.env` 里 App ID/Secret 是否填对。

**Q: 卡片能发出来，但点按钮报 `200340`？**
A: 优先检查 **事件与回调 → 回调配置** 是否已添加 **卡片回传交互**（`card.action.trigger`），并且订阅方式是 **使用长连接接收回调**。只在"事件配置"里添加 `im.message.receive_v1` 不够。

**Q: 提示找不到 claude 或 codex CLI？**
A: 那个引擎按钮不能用。评分任务可以用 **直接 API 运行**（DeepSeek）；原创 / 改写建议先装 Claude Code 或 Codex CLI。若已安装但仍找不到，在 `%APPDATA%\baokuan-rewrite\.env` 里指定 `CLAUDE_CMD` / `CODEX_CMD`。

**Q: 我家电脑关机了还能用吗？**
A: 不能。这个方案需要电脑持续开机 + Bot 服务常驻。如果你想要纯云端方案，请用飞书自带 webhook，不是这里讲的长连接。

**Q: 不想用飞书了，怎么禁用？**
A: 不启动 `start_bot.bat` 就行。`start_app.bat` 的 Streamlit 网页不依赖飞书，独立工作。
